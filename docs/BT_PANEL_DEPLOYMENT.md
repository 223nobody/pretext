# Baota Panel Deployment

This guide uses Docker Compose for the app runtime and Baota/Nginx for domain, HTTPS, and reverse proxy.

## 1. Server Requirements

- Baota panel with Nginx installed.
- Docker and Docker Compose enabled in Baota or installed on the server.
- A domain resolved to the server, for example `pretext.example.com`.
- Ports `80` and `443` open in the firewall.

## 2. Prepare Environment

Upload the project to a server path such as:

```bash
/www/wwwroot/pretext-reader
```

Create the runtime environment file:

```bash
cd /www/wwwroot/pretext-reader
cp .env.example .env
```

Edit `.env` and replace:

```env
PUBLIC_DOMAIN=pretext.example.com
PUBLIC_ORIGIN=https://pretext.example.com
ALLOWED_ORIGINS=https://pretext.example.com,http://pretext.example.com
VITE_API_URL=/api/v1
```

Keep `VITE_API_URL=/api/v1` for same-domain Baota reverse proxy deployment.

## 3. Start Containers

In the project directory:

```bash
docker compose up --build -d
docker compose ps
```

Expected local service targets:

- Frontend: `http://127.0.0.1:3000`
- Backend: `http://127.0.0.1:8000/api/v1/health`

## 4. Baota Website Setup

In Baota panel:

1. Create a website for your domain.
2. Enable SSL for the domain.
3. Add reverse proxy rules in the site Nginx config.

Use this Nginx server config pattern and replace `pretext.example.com`:

```nginx
server {
    listen 80;
    listen 443 ssl http2;
    server_name pretext.example.com;

    client_max_body_size 50m;

    # Baota may manage these SSL certificate paths automatically.
    # Keep the SSL lines generated by Baota if they already exist.

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
        proxy_send_timeout 120s;
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

If Baota already generated a `server` block, keep its SSL certificate lines and only add the `client_max_body_size`, `location /api/`, and `location /` blocks.

## 5. Health Checks

Check these URLs after deployment:

```text
https://pretext.example.com/healthz
https://pretext.example.com/api/v1/health
```

The backend health response should contain:

```json
{"success":true,"data":{"status":"ok","service":"pretext-reader"}}
```

## 6. OCR Option

Scanned PDF OCR is disabled by default.

To enable it:

```env
ENABLE_OCR=true
OCR_LANGUAGE=eng
```

The backend container or host image must include Tesseract and the selected language data. Keep OCR disabled until that dependency is installed.

## 7. Updating

After pulling new code or changing `.env` values that affect the frontend build:

```bash
docker compose up --build -d
docker compose ps
```

Changing `VITE_API_URL`, `VITE_APP_NAME`, or `VITE_DEFAULT_THEME` requires rebuilding the frontend image.
